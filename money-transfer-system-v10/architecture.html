  @Valid @ModelAttribute TransferRequestDto req,

@ControllerAdvice — Global Exception Handling

Centralized error handling for all controllers

@ControllerAdvice
public class GlobalExceptionHandler {

  @ExceptionHandler(AccountNotFoundException.class)
  public String handle(AccountNotFoundException ex,
      RedirectAttributes ra) {
    ra.addFlashAttribute("errorMessage", ex.getMessage());
    return "redirect:/transfer";
  }

  @ExceptionHandler(InsufficientBalanceException.class)
  // ... same pattern

  @ExceptionHandler(BindException.class)
  // ... extracts field errors

  @ExceptionHandler(Exception.class)
  // ... generic fallback
}

AccountNotFoundException → flash error → redirect to form

InsufficientBalanceException → flash error → redirect to form

BindException → extracts field error messages → redirect

Exception → generic fallback → "unexpected error"

Order matters: Most specific exceptions first, generic Exception last. Spring matches the most specific handler.

@InitBinder — Custom Type Conversion

Why empty string → Double fails, and how to fix it

The Problem

HTML: <input type="number" value="">
Browser sends: amount=
Spring tries: "" → Double
NumberFormatException!

Controller never runs.
@ExceptionHandler(Exception.class) catches it.
User sees: "unexpected error"

The Fix

@InitBinder
public void initBinder(WebDataBinder binder) {
  binder.registerCustomEditor(
    Double.class,
    new CustomNumberEditor(
      Double.class, true));
  // true = allow empty
}

Now: "" → null → @NotNull catches it → "Amount is required"
