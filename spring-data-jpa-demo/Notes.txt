JPA Entity Annotations
========================

@Entity
   - marks a class as a JPA entity (maps to a database table)
   - must have a no-arg constructor (Lombok @NoArgsConstructor or default)

@Table(name = "accounts", schema = "public")
   - specifies the table name and schema
   - without it, table name = class name

@Id
   - marks the primary key field

@Column(name = "customer_name")
   - maps field to a specific column name
   - without it, column name = field name (camelCase → snake_case with Hibernate)

@Enumerated(EnumType.STRING)
   - stores enum as string in DB (vs EnumType.ORDINAL which stores index number)
   - STRING is safer — reordering enums won't break data

@Temporal(TemporalType.DATE)
   - specifies how date/time is stored: DATE, TIME, or TIMESTAMP

@Embeddable / @Embedded
   - @Embeddable on Address — marks it as a reusable component (not a separate table)
   - Address fields are embedded directly into the parent table's columns
   - use @Column on Address fields to control column names

@ElementCollection
   - for collections of @Embeddable or basic types
   - creates a separate table to store the collection items
   - @CollectionTable specifies the join table name and join column


JPA Relationship Mappings
===========================

@ManyToOne
   - many accounts belong to one customer
   - Account has @ManyToOne Customer → foreign key in accounts table
   - @JoinColumn(name = "customer_id") specifies the FK column
   - fetch = FetchType.LAZY → loads customer only when accessed (performance!)

@OneToMany(mappedBy = "customer")
   - one customer has many accounts
   - Customer has @OneToMany List<Account>
   - mappedBy = "customer" → the Account.customer field owns the relationship
   - no extra join table — uses the FK already in accounts table

FetchType:
   - LAZY (default for @OneToMany, @ManyToMany) → loads on first access → better performance
   - EAGER (default for @ManyToOne, @OneToOne) → loads immediately with parent → convenient but can be slow


Spring Data JPA Repository
=============================

JpaRepository<EntityType, IdType>
   - just extend the interface — Spring auto-creates the implementation!
   - built-in methods (no code needed):
     - findAll()          → SELECT * FROM table
     - findById(id)       → SELECT * FROM table WHERE id = ?
     - save(entity)       → INSERT or UPDATE
     - deleteById(id)     → DELETE FROM table WHERE id = ?
     - count()            → SELECT COUNT(*) FROM table
     - existsById(id)     → check if exists

@Query("SELECT a FROM Account a WHERE a.customer.id = :customerId")
   - custom JPQL query (Java Persistence Query Language)
   - JPQL uses entity/field names, not table/column names
   - :customerId → method parameter binding

Query derivation (method name queries):
   - findByName(String name)           → WHERE name = ?
   - findByBalanceGreaterThan(double)  → WHERE balance > ?
   - findByCustomerIdAndBalance(...)   → WHERE customer_id = ? AND balance = ?
   - Spring parses the method name and generates the query!


JPA Configuration (application.properties)
============================================

spring.jpa.hibernate.ddl-auto=update
   - update  → auto-create/alter tables to match entities (dev)
   - create  → drop + create tables on startup
   - create-drop → create on start, drop on shutdown
   - validate → only validate schema, don't change
   - none    → do nothing

spring.jpa.show-sql=true
   - logs all SQL queries Hibernate generates

spring.jpa.properties.hibernate.format_sql=true
   - pretty-print the SQL output


CommandLineRunner
==================
   - @Bean CommandLineRunner → runs code after Spring context is initialized
   - useful for testing/demo: load data, test queries
   - receives String[] args


Our Implementation
====================

   Entity classes:
   - Customer (@Entity, @Id, @Column, @Enumerated, @ElementCollection, @OneToMany)
   - Account (@Entity, @Id, @ManyToOne, @JoinColumn)
   - Address (@Embeddable, @Column)
   - CustomerGender (enum: MALE, FEMALE, OTHER)

   Repository interfaces:
   - CustomerRepository extends JpaRepository<Customer, Long>  → all CRUD for free
   - AccountRepository extends JpaRepository<Account, String>  → CRUD + custom @Query

   Service:
   - AccountService → @Service, constructor injection, @Transactional
   - getAllAccounts() → findAll()
   - getAllAccountsByCustomerId() → custom @Query
