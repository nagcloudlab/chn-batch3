Spring Boot

Goal

Get started quickly with minimal configuration. No XML, no boilerplate @Bean methods for common infrastructure.

Philosophy

Convention over configuration. Spring Boot provides sensible defaults — you override only what you need to customize.

Meta-Annotation
@SpringBootApplication
One annotation replaces three — combines @Configuration + @ComponentScan + @EnableAutoConfiguration

What it combines

@Configuration
This class is a Spring configuration class
@ComponentScan
Scan current package and sub-packages for @Component classes
@EnableAutoConfiguration
Enable Spring Boot auto-configuration based on classpath

v6 Main Class

// @Configuration
// @ComponentScan(basePackages = "com.npci")
// @EnableAutoConfiguration
@SpringBootApplication @EnableAspectJAutoProxy
@EnableTransactionManagement
public class MoneyTransferSystem {

  public static void main(String[] args) {
    ConfigurableApplicationContext ctx =       SpringApplication.run(         MoneyTransferSystem.class, args);   }
}
Auto-Configuration
Auto-Configuration Magic
Spring Boot detects libraries on classpath and auto-configures beans for you

v5 — Manual @Bean methods (MtsConfiguration.java)

@Bean public DataSource dataSource() {   HikariConfig config = new HikariConfig();   config.setJdbcUrl(dbUrl);   config.setUsername(dbUser);   config.setPassword(dbPassword);   return new HikariDataSource(config); }
@Bean public JdbcTemplate jdbcTemplate() {   return new JdbcTemplate(dataSource()); }
@Bean public PlatformTransactionManager txMgr() {   return new DataSourceTransactionManager(dataSource()); }

~30 lines of boilerplate configuration code

v6 — Just application.properties

# application.properties spring.datasource.url=jdbc:postgresql://... spring.datasource.username=postgres spring.datasource.password=mysecretpassword spring.datasource.hikari.maximum-pool-size=20
// Spring Boot auto-creates:
// DataSource (HikariDataSource)
// JdbcTemplate
// TransactionManager

// NO MtsConfiguration class needed!
// NO @Bean methods needed!

4 lines of properties replace ~30 lines of Java config

Entry Point
SpringApplication.run()
One line replaces four — creates context, auto-configures, starts server

v5 — Manual context setup

AnnotationConfigApplicationContext ctx
  = new AnnotationConfigApplicationContext();

ctx.register(MtsConfiguration.class); ctx.getEnvironment().setActiveProfiles("dev"); ctx.refresh();

4 lines: create + register + profiles + refresh

v6 — One-liner

ConfigurableApplicationContext ctx =   SpringApplication.run(     MoneyTransferSystem.class, args);
// Does everything:
// 1. Creates ApplicationContext
// 2. Runs auto-configuration
// 3. Component scan from package
// 4. Starts embedded server (if web)

1 line does it all

The Big Picture
The Complete Journey — v1 to v6
What each version taught us

v1 — The Problems

Tight coupling, new inside classes.
Why DI matters.

v2 — Manual DI

Interfaces + Factory + Constructor Injection.
