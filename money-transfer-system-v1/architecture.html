The Problem in Code

public void transfer(double amount, String from, String to) {

    // ❌ Problem: creating dependency inside the method
    JdbcAccountRepository repo = new JdbcAccountRepository();

    Account fromAcc = repo.loadAccount(from);
    Account toAcc = repo.loadAccount(to);
    // ... transfer logic ...
}

Problems

1. Tight Coupling (Design Issue)

TransferServiceImpl directly depends on JdbcAccountRepository.
Cannot swap to JPA, MongoDB, or mock without changing service code.
Unit testing is difficult — can't isolate the service layer.

2. Wasteful Object Creation (Performance Issue)

Every transfer() call creates a new JdbcAccountRepository.
Wastes memory, connections, and increases latency.

3. Root Cause

The dependent (TransferServiceImpl) creates its own dependency using new.
It controls what to use and when to create — no flexibility.
