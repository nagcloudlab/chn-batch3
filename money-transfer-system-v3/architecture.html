Dependency Injection — Constructor vs Setter

Constructor Injection (Required)

@Service("transferService")
public class TransferServiceImpl {

  private AccountRepository repo;

  // constructor injection — required dep   public TransferServiceImpl(     @Qualifier("jdbcAccountRepository")     AccountRepository repo) {     this.repo = repo;   }
}

✓ Immutable   ✓ Required at creation   ✓ No @Autowired needed (single constructor)

Setter Injection (Optional)

  private NotificationService notification;

  @Autowired(required = false)   public void setNotification(     NotificationService notification) {     this.notification = notification;   }

// usage: null-safe check
if (notification != null) {
  notification.sendNotification(...);
}

✓ Optional dependency   ✓ Can be absent at runtime   ✓ Needs @Autowired

Key Concepts in v3

@ComponentScan

Tells Spring to scan packages and auto-discover classes annotated with @Component and its specializations.

Stereotype Annotations

@Repository, @Service, @Controller, @Configuration — all specializations of @Component with semantic meaning.

@Autowired

Tells Spring to inject a dependency. Not needed on single constructor. Use required=false for optional deps.

@Qualifier

When multiple beans match an interface, @Qualifier("name") tells Spring which specific bean to inject.

@Bean

Used in @Configuration classes to manually define beans (like DataSource) that can't be annotated with @Component.

Mixed Config

@ComponentScan for your classes + @Bean for third-party libs (HikariCP). Best of both worlds.

Inversion of Control (IoC)

In v2, your main() created and wired objects manually.
In v3, Spring scans annotations and does it automatically — you just declare intent.
