Dependency Injection
Constructor vs Setter Injection
Required deps via constructor, optional deps via setter

Constructor Injection (Required)

@Service("transferService")
public class TransferServiceImpl {

  private AccountRepository repo;

  // no @Autowired needed (single constructor)   public TransferServiceImpl(     @Qualifier("jdbcAccountRepository")     AccountRepository repo) {     this.repo = repo;   }
}

✓ Immutable   ✓ Required at creation   ✓ No @Autowired needed

Setter Injection (Optional)

  private NotificationService notif;

  @Autowired(required = false)   public void setNotification(     NotificationService notif) {     this.notif = notif;   }

// null-safe usage
if (notif != null) {
  notif.sendNotification(...);
}

✓ Optional dep   ✓ Can be absent   ✓ Needs @Autowired

Configuration
@Value & SpEL Expressions
Inject properties, system values, and computed expressions

Property Placeholders ${...}

// from application.properties
@Value("${spring.datasource.url}") private String dbUrl;

// with default value
@Value("${spring.datasource.hikari.maximum-pool-size:10}") private int maxPoolSize;

Loaded via @PropertySource("classpath:application.properties")

SpEL Expressions #{...}

// call static methods
@Value("#{T(java.lang.Runtime).getRuntime().availableProcessors()}") int numberOfCpuCores;

// read system properties
@Value("#{systemProperties['user.home']}") private String userHome;

// read from another bean
@Value("#{mtsConfig.dbUrl}") private String dbUrl;
Environment
@Profile & @Conditional
Control which beans are created based on environment or runtime conditions

@Profile — Environment-Based

@Repository("jdbcAccountRepository")
@Profile("dev") public class JdbcAccountRepository { ... }

@Repository("jpaAccountRepository")
@Profile("prod") public class JpaAccountRepository { ... }

// activate in main()
ctx.getEnvironment().setActiveProfiles("dev");

dev → JDBC  |  prod → JPA

@Conditional — Custom Logic

@Bean
@Conditional(PostgresDriverCondition.class) public DataSource dataSource(...) { ... }

// the condition class
public class PostgresDriverCondition
  implements Condition {
  public boolean matches(...) {
    Class.forName("org.postgresql.Driver");
    return true;
  }
}

Bean created only if PostgreSQL driver is on classpath

Summary
Spring Annotations Covered
All annotations used in v3 at a glance

@Bean

Define beans manually in @Configuration for third-party libs

@ComponentScan

Auto-discover @Component classes in specified packages

@Autowired

Inject dependency. Not needed on single constructor

@Qualifier

Pick specific bean when multiple match an interface

@Primary

Default bean when no @Qualifier specified

@Value

Inject properties & SpEL expressions into fields

@Profile

Activate beans only for specific environments (dev/prod)

@Conditional

Custom runtime condition to decide bean creation

@PostConstruct

Init logic after bean is created and wired

@PreDestroy

Cleanup logic before bean is destroyed

@EventListener

React to events without coupling to publisher

@Lazy / @Scope

Control instance timing and multiplicity

Inversion of Control (IoC)

In v2, your main() created and wired objects manually.
In v3, Spring scans annotations and does it automatically — you just declare intent.

What v3 Gains Over Manual Wiring

No XML
Config lives next to the code it configures
Type-Safe
Compiler catches errors, not XML parser
Auto-Discovery
@ComponentScan finds beans automatically
@Qualifier
Precise control over which impl to inject
Required vs Optional
Constructor DI + @Autowired(required=false)
Event-Driven
@EventListener for loose publisher-listener coupling
